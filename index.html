Design framework and boilerplate

Build on industry standards like ITCSS, BEMit and SMACSS.

Design framework

Chopstick is a mobile-first responsive framework written in Sass. It uses the ITCSS architecture and BEMit naming methodology. It is build to enable you to quickly create custom design systems.

Requirements

  • Bower
  • Sass

Get started

Step 1 git clone git@github.com:getchopstick/chopstick.git

Step 2 bower install

Boilerplate

The Chopstick boilerplate is a prototyping workflow build around Jekyll, Gulp and the Chopstick framework. It is build to enable you to quickly create high-fidelity prototypes, marketing sites and blogs.

Requirements

  • Bower
  • Gulp
  • Jekyll

Get started

Step 1 git clone git@github.com:getchopstick/chopstick-boilerplate.git

Step 2 bundle install && npm install && bower install

Boilerplate documentation

Why?

Scalable

We use a single-depth-class-based architecture. This makes our framework very robust and easily scalable.

Highly composable

We avoid reliance between elements. Adding is always done by extension, not modification.

Keep it simple

When in doubt we go with the option that is the most resilient, even if it is the most boring one.
